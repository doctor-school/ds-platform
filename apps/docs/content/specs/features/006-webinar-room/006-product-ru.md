> Эпик: [«Вебинары» — продуктовый бриф](../../product/webinars/brief-ru.md) · Волна 1, утверждённый вариант A «тонкая вертикаль». **MVP-критичная поверхность для живого вебинара 2026-07-17.**

## Суть фичи

Комната, где зарегистрированный врач собственно смотрит вебинар: встроенный внешний стрим (Rutube или аналог — мы **не** строим ни транскодинг, ни медиасервер), живой чат и невидимая для врача фиксация присутствия. Доступ **закрыт на сервере** — только аутентифицированные **и** зарегистрированные; никакой мягкой UI-стены поверх плеера в легаси-стиле. Плеер выбирается **явным enum провайдера** (`rutube | youtube | vk | cdnvideo`), настраиваемым в админке — никогда не URL-sniffing. Чат едет на Centrifugo (уже в нашем стеке). Присутствие фиксируется с первой минуты через **server-authoritative heartbeat** — комната шлёт аутентифицированный сигнал каждые N секунд, бэкенд дописывает его в долговечную append-таблицу Postgres, — потому что минуты присутствия по каждому врачу и есть результат для спонсора; по первому вебинару отчёт спонсору — ручная выгрузка из этих данных.

Состав комнаты для вебинара 2026-07-17: **embed-плеер + живой чат + heartbeat-присутствие**, а в шапке комнаты — врачёвый chrome с канваса: **переключатель светлой/тёмной темы** (за ним — портал-широкий механизм темы; видимый переключатель живёт только здесь до унифицированного ре-скина chrome портала, #510) и **аватар с инициалами**, построенный из настоящего отображаемого имени врача — собираемого **just-in-time** один раз, перед первым входом в комнату (регистрация не трогается). Опросы и вопрос лектору — волна 2.

## Пользовательские истории

- **US-1** — Как **зарегистрированный врач**, когда вебинар в эфире, я вхожу в комнату со страницы мероприятия (или из «мои события») и смотрю стрим без единого препятствия по вине платформы — легаси-класс «не смог подключиться к эфиру» списан.
- **US-2** — Как **врач в комнате**, я читаю и пишу сообщения в живом чате рядом со стримом и вижу чужие сообщения без перезагрузки страницы.
- **US-3** — Как **врач**, моё присутствие в комнате фиксируется автоматически всё время просмотра — я никогда ничего не нажимаю, чтобы «доказать», что я здесь (виджеты подтверждения присутствия — механика НМО волны 2, не требование волны 1).
- **US-4** — Как **фарм-спонсор**, после вебинара я получаю фактические минуты присутствия по каждому врачу (с контактами) для своего отчёта — по первому вебинару это ручная выгрузка из зафиксированных данных.
- **US-5** — Как **неаутентифицированный или незарегистрированный посетитель**, я не могу добраться до содержимого комнаты; вместо этого меня ведут залогиниться / зарегистрироваться на мероприятие — и это принуждает сервер, а не прячет UI.
- **US-6** — Как **врач в комнате**, я переключаю комнату между светлой и тёмной темой переключателем в шапке комнаты; пока я не выбрал — портал следует настройке моей системы, а как только выбрал — мой выбор сохраняется между перезагрузками и визитами.
- **US-7** — Как **врач**, перед моей первой вебинарной комнатой меня **один раз** спрашивают моё имя («Имя и фамилия»); после этого шапка комнаты показывает аватар с инициалами моего настоящего имени, и этим же именем подписаны мои сообщения в чате эфира — его видят другие участники (промпт заранее это раскрывает). Регистрация остаётся такой же короткой, как сегодня.

## Флоу

**Happy path — смотрю вживую (US-1, US-2, US-3):**

1. Режиссёр открывает комнату (фича 007); мероприятие в состоянии `live`.
2. Зарегистрированный врач идёт по пути присоединения со страницы мероприятия / из «мои события» → комната рендерится: встроенный плеер (сконфигурированный провайдер) + живой чат.
3. Пока врач в комнате, клиент шлёт аутентифицированный heartbeat каждые N секунд; каждый сигнал ложится в append-only-таблицу присутствия с врачом, мероприятием и временем.
4. Врач общается в чате; сообщения разлетаются в реальном времени всем в комнате.

**Первый вход — отображаемое имя (US-7):**

1. Зарегистрированный врач, чьё отображаемое имя ещё не задано, идёт по пути присоединения в live-комнату.
2. До рендера комнаты портал **один раз** спрашивает «Имя и фамилия»; текст промпта раскрывает, что имя увидят участники чата эфира («Ваше имя будут видеть участники чата эфира»); пустое или состоящее из пробелов значение отклоняется с правдивой ошибкой.
3. Сохранённое имя ложится на запись пользователя врача через аутентифицированный эндпоинт; комната рендерится с аватаром в шапке, показывающим инициалы настоящего имени.
4. При каждом последующем входе промпт больше не появляется; отправленные врачом сообщения чата подписаны его именем — его видят все участники (у кого имя не задано — fallback на не-PII метку «Участник <tag>»).

**Тема (US-6):**

- Пока врач не сделал выбор, комната следует системной настройке светлого/тёмного; переключатель в шапке меняет тему мгновенно, а явный выбор сохраняется между перезагрузками и визитами и побеждает системную настройку.

**Ветки доступа (US-5):**

- Неаутентифицированный посетитель попадает на комнату → отправляется через auth (003), затем переоценивается.
- Аутентифицированный, но незарегистрированный врач попадает на комнату → его ведут зарегистрироваться (фича 005), затем впускают.
- Мероприятие не `live` (ещё не открыто / уже закрыто) → смотрибельной комнаты нет; врач видит правдивое состояние жизненного цикла (состояния страницы фичи 004).

**После эфира (US-4):**

1. Режиссёр закрывает комнату (фича 007) → heartbeat'ы по мероприятию перестают приниматься.
2. Зафиксированные данные присутствия дают минуты по каждому врачу; оператор передаёт отчёт спонсору (по первому вебинару ручная выгрузка допустима).

## Продуктовые критерии приёмки

- Доступ в комнату **принуждается на сервере**: содержимое комнаты (конфиг плеера, доступ к чату, приём heartbeat'ов) получает только аутентифицированный зарегистрированный врач. Прямой URL, пересланная ссылка или сконструированный запрос не обходят ворота.
- Плеер инстанцируется из **явного enum провайдера**, хранимого в стрим-конфиге мероприятия; значения провайдера — закрытый список (`rutube | youtube | vk | cdnvideo`) — никогда не выводятся из строки URL.
- Когда встроенный стрим **не начинает воспроизведение** (застой CDN провайдера, видео с запрещённым встраиванием), врач видит **правдивый статус в комнате**, комната **автоповторяет** и предлагает кнопку **«Перезапустить плеер»** — никогда не молчаливый чёрный экран, никогда не перезагрузку всей страницы и никогда не ссылку, уводящую врача с платформы (контроль присутствия в комнате — ключевая функция площадки — решение владельца 2026-07-22). Присутствие продолжает считаться всё время сбоя. (US-1)
- Чат **реального времени**: отправленное сообщение доходит до остальных участников комнаты без перезагрузки (транспорт Centrifugo); чат доступен, пока комната открыта.
- Heartbeat'ы присутствия **server-authoritative и долговечны**: аутентифицированные сигналы дописываются в таблицу Postgres с точностью, достаточной для вычисления фактических минут присутствия по каждому врачу; параллельные вкладки не завышают минуты врача.
- Зафиксированных данных достаточно, чтобы передать спонсору первого вебинара отчёт с минутами присутствия по каждому врачу через **ручную выгрузку** — UI отчётов в волне 1 не требуется.
- Шапка комнаты несёт **переключатель светлой/тёмной темы** — иконку-кнопку 44×44 по канвасу: дефолт следует системной настройке, явный выбор побеждает и сохраняется между перезагрузками и визитами, а страницы рендерятся в разрешённой теме с первой отрисовки — никогда вспышки не той темы. Переключатель отгружается **только** в шапке вебинарной комнаты; его размещение на остальных поверхностях портала отложено до унифицированного ре-скина chrome портала (#510).
- Перед первым входом в вебинарную комнату врача **один раз** спрашивают «Имя и фамилия»; пустое/пробельное значение отклоняется; регистрация не собирает ничего нового. Аватар в шапке показывает инициалы настоящего сохранённого имени — никогда инициалы, сфабрикованные из email или заглушки, — а имя отдаётся только самому врачу.
- 2026-07-17: реальные врачи смотрят эфир в этой комнате end-to-end с **нулём жалоб «не смог подключиться» по вине платформы**.

## Вне scope

- Опросы, вопрос лектору, виджеты подтверждения присутствия / «титровальные объекты» (волна 2).
- Автоначисление НМО (90 минут + 2 подтверждения) и авто-отчёт спонсору «Отчёт партнёра V2» (волна 2) — волна 1 только фиксирует данные, которые им понадобятся.
- Видеотранскодинг, собственный медиасервер, DRM / подписанное воспроизведение внешнего стрима, телеметрия уровня плеера (вне embed-рамки эпика).
- Архив записей (волна 2).
- Режиссёрские органы открытия/закрытия комнаты и стрим-конфиг (фича 007).
- Размещение переключателя темы на поверхностях портала вне шапки вебинарной комнаты — отложено до унифицированного ре-скина chrome портала (#510); сам механизм темы — портал-широкий с первого дня.
- Сбор имени врача на регистрации (воронка sign-up не трогается — решение владельца, ноль добавленного трения на живом проде). **Больше не out:** показ имени врача другим участникам чата — теперь в scope (решение владельца 2026-07-23, Вариант A): имя автора подписывает его же сообщения в живом чате, с fallback на не-PII метку участника, когда имя не задано.

## Открытые вопросы

- **Расхождение по плееру** (recon §5, §10-1): _разрешено._ Enum провайдера зафиксирован закрытым списком `rutube | youtube | vk | cdnvideo` — RU-достижимые, встраиваемые провайдеры (Rutube — стандартный дефолт; #1154). SDN Player не в списке; расширение позже — аддитивная миграция.
- **Каденс heartbeat** (recon §10-3): легаси-экспорт показывает пинги 60 с, но продакшен может отличаться; подтвердить живой каденс до фиксации N и арифметики «1 сигнал = N секунд».
- **Артефакт сертификата** (recon §10-5, волна 2): только баллы НМО или скачиваемый PDF сертификата? Решение владельца до НМО-фичи волны 2.
- **Точная форма отчёта партнёра V2** (recon §10-9, волна 2): подтвердить точные колонки/джойны с владельцем до построения авто-отчёта; ручной выгрузке волны 1 нужны только сырые минуты.
