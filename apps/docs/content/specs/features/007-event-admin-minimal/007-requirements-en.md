# 007 — Minimal event admin: create/edit, stream config, room control, lifecycle (Requirements)

> Epic: [Webinars — product brief](../../product/webinars/brief.md) · Wave 1, approved variant A «thin vertical». PRD source: [`007-product.md`](./007-product.md) (US-1…US-5). **No mockup canvas exists for the admin surface (known Stage-A gap, PRD «Approved mockup»)** — wave 1 builds on **stock Refine UI** (ADR-0004); the owner checkpoint (keep stock vs commission an admin canvas) is recorded below.

## Outcomes

- An **operator** creates an event ≥1 month ahead in the `admin` app with the field set the public page (004) needs — title, school/series, **date + time explicitly in МСК** (one canonical instant stored), description, speakers, target specialties, a program **PDF**, and a sponsor/partner reference — and keeps it in `draft` until it is ready to announce (US-1).
- An **operator** edits a published event as details settle — in particular **replaces the program PDF** (it "often changes") — and the public event page (004) reflects the edit, serving the **current** file (US-2).
- An **operator/director** configures the stream by pasting the link and **picking the provider explicitly from a closed enum** (`rutube | youtube`) plus an embed reference, so the room (006) instantiates exactly the right player with **no URL guessing**; a wrong URL is correctable while `published` (US-3).
- A **director** **opens the live room** when the broadcast starts and **closes it** when it ends — those two actions are the `published → live` / `live → ended` transitions that admit registered viewers (006) and bound the presence capture (US-4).
- An **operator** moves the event along **one closed lifecycle** — `draft → published → live → ended → archived` — with the admin UI offering **only the transitions valid from the current state**, and the server refusing any invalid jump; there are **no boolean flags to reconcile** (US-5).
- **What admin shows as the event's state is exactly what the portal surfaces reflect** — listing visibility (004), page state (004), registration availability (005), room access (006) — one `EventLifecycleState` source of truth, no drift (US-1…US-5).
- The **minimal path — create → publish → configure stream → open → close — is completable by one operator/director for the 2026-07-17 webinar without developer intervention** (US-1…US-5, PRD acceptance).

## Lead authoring decisions (owner review pending)

The PRD left three questions open for the owner; the owner was unavailable at authoring time, so each is settled here as a **lead decision (2026-07-06, owner review pending)** and can be vetoed at spec review. Each is also carried inline on the EARS/Constraint it shapes.

- **LD-1 — Speaker representation = ordered free-text entries.** Wave 1 stores speakers as an **ordered list of free-text entries** (name + regalia/credentials strings), validated as text only. **References to real user/speaker records are deferred to wave 2**, bundled with the deferred speaker-directory management — a ref without a directory has nothing to point at. The schema keeps the field an **extensible list of entries** so a wave-2 ref variant is an additive migration, not a reshape (LD-1; EARS-1, Scope → Out of scope).
- **LD-2 — `ended → archived` is a manual operator action.** Wave 1 has **no scheduler and no time-based automation**: archiving is an explicit operator command (legacy parity). A **time-based auto-archive** policy is a named **wave-2** candidate (LD-2; EARS-6, Scope → Out of scope).
- **LD-3 — Admin access = one trusted admin group (`platform_admin`).** Operator and director are the **same platform role** in wave 1 — the single admin-app group `platform_admin` from the ADR-0001 §2 IdP model. **Distinct operator vs director roles and manager-scoped (owner-of-record) event lists are wave 2** (LD-3; EARS-8, Constraints).

## Scope

**In:**

- **Create event (→ `draft`)** — an operator creates an event in the admin app carrying: title, school/series, **date + time in МСК** stored as one canonical instant, duration, description, **speakers as ordered free-text entries** (LD-1), target specialties, a **program PDF** uploaded to object storage, and a sponsor/partner reference. A new event starts in `draft` and is not publicly reachable until published (EARS-1).
- **Edit event + replaceable program PDF** — an operator edits an event's fields at any pre-archive state; **replacing the program PDF after publish** updates the stored object reference and the public event page (004) serves the **current** file — the previous file is not served after replacement (EARS-2).
- **Stream configuration (explicit provider enum + embed ref)** — an operator/director sets the event's stream config to a provider from the **closed enum `rutube | youtube`** plus an **embed reference** (the provider-scoped stream id, never a URL to be sniffed); the config is correctable while `published`, and the room (006) consumes exactly this config (EARS-3).
- **Publish transition (`draft → published`)** — an operator publishes an event; on publish it becomes publicly reachable on the 004 listing + page and **registration opens** (005). Publish is refused unless the event is in `draft` (EARS-4).
- **Open room / close room (`published → live` / `live → ended`)** — a director **opens the room** (the `published → live` transition), which admits registered doctors into the 006 room and starts presence capture; and **closes the room** (the `live → ended` transition), which stops admission and heartbeat/chat acceptance and bounds the presence window (EARS-5).
- **Archive transition (`ended → archived`)** — an operator **manually** archives an `ended` event (LD-2); an archived event leaves all public surfaces (the 004 archived notice / listing drop). No scheduler (EARS-6).
- **The single event state machine** — one `EventLifecycleState` field with the **closed transition set** `draft→published`, `published→live`, `live→ended`, `ended→archived`; the admin UI **offers only the transitions valid from the current state**, and the server **refuses any invalid jump** (e.g. `draft→live`, `published→ended`, reopening an `archived` event, any backward move) — enforced server-side, not merely hidden in the UI (EARS-7).
- **Admin read side + the produced read models** in `apps/api` — the operator's admin list/detail reads (`EventAdminList`, `EventAdminDetail`) plus the **produced** projections the rest of the epic consumes (`EventLifecycleState`, the 004 `PublicEventPage` / `UpcomingBroadcastCard`, the 006 stream config), all classified **`authenticated` `platform_admin` `fast-path`** in the endpoint-authz matrix (ADR-0001 §2) for the admin reads/writes (the produced public projections keep 004's `public` classification) (EARS-8).
- **Admin/portal single source of truth** — the same `EventLifecycleState` the 007 commands write drives the admin display **and** the public surfaces (004/005/006); there is no second visibility flag and no drift (EARS-9).
- **RU admin copy from the typed message catalog** — no hardcoded user-facing strings on the admin surface, reusing the i18n-ready structure established in 003 (EARS-21) and 004/005/006; any absolute date/time presented in `Europe/Moscow` labeled **МСК**, computed from the canonical instant (EARS-10).
- **Stock-Refine admin surface** — built on **stock Refine UI** (ADR-0004) with the adopt-before-bespoke gate; ADR-0013 token discipline applies where any bespoke styling appears. **No canvas exists** for this surface (Stage-A gap) → **no** canvas-fidelity EARS; the owner checkpoint (keep stock vs commission an admin canvas) is named (EARS-11).

**Explicitly out** (each a named deferral, not a silent default):

- **Speaker directory / real-record references** — wave 1 stores speakers as ordered free-text entries only (LD-1); references to real user/speaker records, per-event speaker cards with ratings, and speaker-directory management are **wave 2**.
- **Time-based / scheduled auto-archive** — `ended → archived` is a manual operator action in wave 1 (LD-2); a time-based auto-archive policy is a wave-2 candidate. There is no scheduler in 007.
- **Distinct operator vs director roles + manager-scoped lists** — wave 1 is one trusted `platform_admin` group (LD-3); a role hierarchy (operator/director split, manager/owner-of-record scoping, legacy RLS branches) is wave 2.
- **Widget / «титровальные объекты» authoring, polls console, question-to-lecturer moderation** — wave 2 (the 006 room defers the same affordances).
- **Program constructor** — the legacy 11-tab editor's program builder is **not** rebuilt; the real workflow is a single final-PDF upload (PRD; recon).
- **Reporting UI / report export buttons** — the first webinar's sponsor report is a **manual export** from the 006 presence data; 007 ships no report UI.
- **NMO configuration** (codes, points, accreditation) — wave 2.
- **Unpublish / `published → draft`** — the PRD does **not** name an unpublish transition, so 007 does **not** invent one; the closed transition set has no backward move (EARS-7). A correction to a `published` event is an edit (EARS-2) or a stream-config fix (EARS-3), not a state reversal.
- **Registration, the room, the public page/listing themselves** — 007 **authors** the event aggregate + lifecycle + stream config; the registration write (005), the room admission/chat/presence (006), and the public page/listing (004) are **consumers**, owned by those features.
- **Congress landing blocks, offline events, paid flows** — out of the epic.

## Constraints

- **One state machine, one closed transition set — server-enforced.** Lifecycle is a single `EventLifecycleState` enum (`draft → published → live → ended → archived`), replacing the legacy boolean scatter (`draft` / `published?` / `archive` / `visible_in_rg` / …) that made "is this event visible?" ambiguous (recon §7d). The **only** legal transitions are the four forward moves; every other move (backward, skip, reopen-archived) is **refused server-side**, and the admin UI **offers only** the transitions valid from the current state. Hiding an invalid transition in the UI is necessary but **not** sufficient — the server is the authority (EARS-7).
- **Admin state is the single source of truth.** The state 007 writes is the state 004 (listing/page), 005 (registration gating), and 006 (room access) read; there is **no** second visibility flag to reconcile and **no** drift (EARS-9, PRD acceptance).
- **Stream provider is an explicit enum, never URL-sniffing.** Stream config is `{ provider ∈ {rutube, youtube}, embedRef }`; the provider is chosen explicitly by the operator and the embed reference is the provider-scoped stream id — the room (006) never infers the provider from the URL string (the legacy mistake, recon §5). Extending the enum beyond `rutube | youtube` later is an additive **migration**, not a shape 007 pre-builds (mirrors the 006 owner decision 2026-07-06).
- **The program PDF lives in object storage and is replaceable.** The program PDF is uploaded to the platform's object storage (Timeweb Object Storage in prod; MinIO on the dev stand, ADR-0003 §, `.claude/rules/dev-stand.md`) — never committed to the repo or embedded in the record — and is **replaceable after publish**: a replacement updates the stored reference so the 004 public page serves the current file, and the superseded file is no longer served (EARS-2, US-2).
- **Canonical Moscow time.** The operator enters a date + time understood as **МСК**; the system stores **one canonical instant** and every absolute date/time rendered in the admin surface (and produced for 004/005/006) is presented in `Europe/Moscow` labeled **МСК**, never drifting to the operator's local timezone (mirrors 004 EARS-12 / 005 EARS-11 / 006 EARS-10).
- **Admin access is one trusted group (`platform_admin`), role fast-path.** The admin commands and admin reads require the `platform_admin` role (LD-3); in wave 1 the check is a **role fast-path** (`auth_check: fast-path`) — the role alone is sufficient because there is one trusted group with no per-event owner scoping. The lifecycle transitions and stream-config writes are **never** callable by `doctor_guest` or a public caller. Admin mutations are the **high-stakes** tier (ADR-0001 §2.5/§8) — the Refine admin app is 2FA-gated (ADR-0004 §3) and the state-changing commands ride the introspection validation path; wave 1 adds no bespoke per-action step-up beyond that posture (EARS-8).
- **007 reuses the shipped auth (003), adds no auth primitive.** The admin principal is a `platform_admin` session issued by the shipped 003/IdP layer; 007 introduces no new credential, session, or token path (ADR-0001; 003 Constraints).
- **No hardcoded origin / URL / storage endpoint.** The admin origin, API endpoints, object-storage endpoint/bucket, and the stream reference are read from configuration / the event record, never hardcoded in code or spec (AGENTS.md §9; `.claude/rules/dev-stand.md`).
- **Stock Refine, adopt-before-bespoke.** The admin surface uses **stock Refine components** (ADR-0004); the `build-ui-from-design-system` gate runs before any bespoke element, and ADR-0013 token discipline (no arbitrary Tailwind) applies where custom styling appears. **There is no admin canvas** (Stage-A gap), so — unlike 004/005/006 — 007 carries **no** canvas-fidelity requirement; the owner decides at the next Stage-A checkpoint whether to keep stock or commission a canvas (EARS-11).
- **Stack** (ADR-0002 / ADR-0004): the event-authoring commands, lifecycle transitions, and admin/produced reads are NestJS + `nestjs-zod`, schema SSOT in `packages/schemas/`, URI versioning `/v1/...`, Vitest + supertest; the admin app is Refine (client-side) on the NestJS API via custom data/auth/access providers. Service-dependent tests `skipIf` their dependency env (`DATABASE_URL`, `IDP_ISSUER`, `S3_ENDPOINT`) is absent so they do not redden the shared CI unit job.

## Event Model

Feature 007 is the **authoring vertical** — the **write side** of the webinar aggregate and the **producer** of the state the read-side slices consume. It owns the event aggregate, the `stream_config`, the program-PDF reference, and the single `EventLifecycleState` state machine. It **produces** `EventLifecycleState`, the 004 `PublicEventPage` / `UpcomingBroadcastCard` projections, and the 006 stream config; it **consumes** only the shipped 003 auth (the `platform_admin` principal) and object storage (the PDF). It owns **no** registration (005), **no** room admission/chat/presence (006), and **no** public rendering (004).

### Commands (handled by `apps/api`, `platform_admin`-authenticated, role fast-path)

- `CreateEvent(fields)` — create an event in `draft` with the full field set (title, school, canonical `startsAt` from a МСК entry, duration, description, ordered free-text speakers, specialties, program-PDF reference, sponsor/partner). Emits `EventCreated`.
- `UpdateEvent(idOrSlug, fields)` — edit any field of a pre-archive event, **including replacing the program PDF** (a new upload supersedes the stored reference). Emits `EventUpdated`.
- `ConfigureStream(idOrSlug, { provider ∈ {rutube, youtube}, embedRef })` — set/correct the stream config; correctable while `published`. Emits `StreamConfigured`.
- `PublishEvent(idOrSlug)` — `draft → published`; the event becomes publicly reachable (004) and registration opens (005). Refused unless `draft`. Emits `EventPublished`.
- `OpenRoom(idOrSlug)` — `published → live`; admits registered doctors into the 006 room and starts presence capture. Refused unless `published`. Emits `EventWentLive`.
- `CloseRoom(idOrSlug)` — `live → ended`; stops room admission + heartbeat/chat acceptance and bounds the presence window (006 EARS-7). Refused unless `live`. Emits `EventEnded`.
- `ArchiveEvent(idOrSlug)` — `ended → archived`; the event leaves all public surfaces. **Manual** (LD-2). Refused unless `ended`. Emits `EventArchived`.

Every transition command validates the **current** state and refuses an invalid jump (EARS-7); the command surface offers no transition the current state does not permit.

### Events

| Event                           | Owner (007) | Notes                                                                                                                                                         |
| ------------------------------- | ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `EventCreated` / `EventUpdated` | `apps/api`  | The authored/edited aggregate (incl. the program-PDF reference). A replacement PDF supersedes the stored reference; 004 serves the current file (US-1, US-2). |
| `StreamConfigured`              | `apps/api`  | The `{ provider, embedRef }` the 006 room instantiates the player from; correctable while `published` (US-3).                                                 |
| `EventPublished`                | `apps/api`  | `draft → published`. Consumed read-only by 004 (public reachability + listing) and 005 (registration opens). One `audit_ledger` row.                          |
| `EventWentLive` (room opened)   | `apps/api`  | `published → live`. Consumed by 006: the room opens (admission + presence capture start) and by 004 (the "live now" signal). One `audit_ledger` row.          |
| `EventEnded` (room closed)      | `apps/api`  | `live → ended`. Consumed by 006: the room closes (beats + posts refused, presence window bounded) and by 004 (ended state). One `audit_ledger` row.           |
| `EventArchived`                 | `apps/api`  | `ended → archived` (manual, LD-2). Consumed by 004: the event leaves public listings; a direct link renders the archived notice. One `audit_ledger` row.      |

### Read models

- **`EventAdminList`** — the operator's list of **all** events regardless of state (`draft`…`archived`), each with its current `EventLifecycleState`, air date/time (МСК), and stream-config completeness — the admin app's index. `platform_admin`-authenticated; never public.
- **`EventAdminDetail`** — the full editable aggregate for one event (all fields + stream config + program-PDF reference + the valid next transitions from the current state). `platform_admin`-authenticated.
- **`EventLifecycleState`** (produced) — the single enum `draft | published | live | ended | archived`; the source of truth every 004/005/006 surface reads.
- **`PublicEventPage` / `UpcomingBroadcastCard`** (produced; **surfaced** by 004) — the publish-safe projection of the authored aggregate (incl. the current `programPdfUrl`); 004 owns the public reads/rendering, 007 owns the authored data behind them.
- **stream config** (produced; **consumed** by 006) — `{ provider ∈ {rutube, youtube}, embedRef }`; the room instantiates the player from it, never from the URL.

### Policies

- **On any transition command** → validate the current state and apply **only** a legal forward move (`draft→published→live→ended→archived`); refuse an invalid jump server-side, offering in the admin UI only the transitions the current state permits (EARS-7).
- **On `PublishEvent`** → the event becomes publicly reachable (004) and registration opens (005) — the single visibility signal, no boolean flag (EARS-4, EARS-9).
- **On `OpenRoom` / `CloseRoom`** → the 006 room's open window is exactly the `live` state; opening starts admission + presence capture, closing stops them and bounds the presence window (EARS-5; 006 EARS-7).
- **On `UpdateEvent` with a new program PDF** → the stored reference is replaced and the 004 page serves the current file; the superseded file is no longer served (EARS-2).
- **On every command/read** → require `platform_admin`; never serve to `doctor_guest` or a public caller; append one `audit_ledger` row per lifecycle transition (EARS-8; ADR-0003 §6).

## EARS requirements

> **Numbering convention:** flat (`EARS-1`, `EARS-2`, …) per ADR-0006 §4. EARS-1…7 are the functional handlers (each becomes a child Issue); EARS-8…11 are cross-cutting ubiquitous / unwanted-behavior requirements enforced across the surface. Each clause carries a `realizes: US-N` backlink to the PRD story it formalizes (ADR-0014 §2).

**Event authoring**

- **EARS-1** _(realizes: US-1)_ — When an operator submits the create-event form in the admin app, the system shall create an event in **`draft`** carrying title, school/series, a **date + time entered as МСК and stored as one canonical instant**, duration, description, **speakers as an ordered list of free-text entries (name + regalia strings), validated as text only** (LD-1 — real-record references are wave 2, and the field stays an extensible list so the wave-2 variant is an additive migration), target specialties, a **program PDF uploaded to object storage**, and a sponsor/partner reference; a `draft` event is **not** publicly reachable (004 EARS-6) until it is published.
- **EARS-2** _(realizes: US-2)_ — When an operator edits an event, the system shall persist the change to the authored aggregate and surface it on the public event page (004); in particular, **replacing the program PDF after publish** shall update the stored object reference so the public page serves the **current** file, and the superseded file shall **no longer be served** — the operator never has to unpublish to correct a detail.
- **EARS-3** _(realizes: US-3)_ — When an operator/director configures the stream, the system shall record a stream config of an **explicit provider from the closed enum `rutube | youtube`** plus an **embed reference** (the provider-scoped stream id, never a URL to be sniffed), correctable while the event is `published`; the 006 room shall instantiate the player from **exactly this config** and never infer the provider from the URL string.

**Lifecycle transitions**

- **EARS-4** _(realizes: US-5)_ — When an operator publishes a `draft` event, the system shall transition it **`draft → published`**, making it publicly reachable on the 004 event page + upcoming listing and **opening registration** (005); publish shall be **refused unless the event is in `draft`**, and the transition shall append one `audit_ledger` row.
- **EARS-5** _(realizes: US-4)_ — When a director **opens the room**, the system shall transition the event **`published → live`** — admitting registered doctors into the 006 room and starting presence capture; and when the director **closes the room**, the system shall transition it **`live → ended`** — stopping room admission and heartbeat/chat acceptance and **bounding the presence window** (006 EARS-7). Open shall be refused unless `published`; close shall be refused unless `live`; each transition appends one `audit_ledger` row.
- **EARS-6** _(realizes: US-5)_ — When an operator archives an `ended` event, the system shall transition it **`ended → archived`** as a **manual** action (LD-2 — no scheduler, no time-based automation in wave 1), after which the event **leaves all public surfaces** (the 004 listing drop + archived notice); archive shall be **refused unless the event is `ended`**, and the transition shall append one `audit_ledger` row.
- **EARS-7** _(realizes: US-5)_ — The event lifecycle shall be a **single `EventLifecycleState` field** with the **closed transition set** `draft→published`, `published→live`, `live→ended`, `ended→archived`; the admin UI shall **offer only the transitions valid from the current state**, and the server shall **refuse any invalid jump** (e.g. `draft→live`, `published→ended`, reopening an `archived` event, or any backward move) — enforcement is **server-side**, not merely hidden in the UI, and there is **no** unpublish / `published→draft` transition (the PRD names none).

**Cross-cutting (ubiquitous / unwanted-behavior)**

- **EARS-8** _(realizes: US-1, US-2, US-3, US-4, US-5)_ — The admin commands (`CreateEvent`, `UpdateEvent`, `ConfigureStream`, `PublishEvent`, `OpenRoom`, `CloseRoom`, `ArchiveEvent`) and the admin reads (`EventAdminList`, `EventAdminDetail`) shall be classified **`access: authenticated`, `required_roles: platform_admin`, `auth_check: fast-path`** in the endpoint-authz matrix (ADR-0001 §2) — one trusted admin group in wave 1 (LD-3), role alone sufficient. **No** lifecycle transition or stream-config/authoring write shall ever be callable by a `doctor_guest` or a public caller; admin mutations ride the high-stakes validation tier (ADR-0001 §2.5/§8) and the admin app is 2FA-gated (ADR-0004). The produced public projections (`PublicEventPage`, `UpcomingBroadcastCard`) keep 004's `public` classification.
- **EARS-9** _(realizes: US-1, US-2, US-4, US-5)_ — The system shall keep **one source of truth for event state**: the `EventLifecycleState` the 007 commands write is exactly what the admin surface displays **and** what 004 (listing/page visibility), 005 (registration gating), and 006 (room access) read — there shall be **no** second visibility flag and **no** boolean scatter to reconcile, so admin and the portal can never disagree about an event's state.
- **EARS-10** _(realizes: US-1, US-2, US-3)_ — The 007 admin surface shall render in **Russian** with **no hardcoded user-facing strings** — all copy (field labels, lifecycle-state names, transition-action labels, validation/error messages, the stream-provider choices) sourced from the typed message catalog established in 003 (EARS-21) and reused in 004/005/006; every **absolute** date/time (the event air date/time on the list and detail) shall be presented in `Europe/Moscow` labeled **МСК**, computed from the stored canonical instant and never drifting to the operator's local timezone.
- **EARS-11** _(realizes: US-1)_ — The admin surface shall be built on **stock Refine UI** (ADR-0004) with the `build-ui-from-design-system` adopt-before-bespoke gate run first; ADR-0013 token discipline (no arbitrary Tailwind values) shall apply to any bespoke styling. Because **no admin canvas exists** (recorded Stage-A gap, PRD «Approved mockup»), 007 carries **no** canvas-fidelity requirement; the choice to keep stock Refine vs commission an admin canvas is an **owner decision at the next Stage-A checkpoint**, named here, not settled by 007.

## Invariants

- Event state is **one** `EventLifecycleState` field with a **closed** transition set; the only legal moves are the four forward transitions, every invalid jump is refused server-side, and the admin UI offers only the currently-valid transitions (EARS-7). There is no unpublish / backward move.
- Admin state **is** portal truth — the state 007 writes is the state 004/005/006 read; there is no second visibility flag and the two can never drift (EARS-9).
- The stream provider is always one of the **closed enum** `rutube | youtube`, chosen explicitly, with a provider-scoped embed reference — never a URL to be sniffed (EARS-3).
- The program PDF lives in **object storage** and is **replaceable**; after replacement the public page serves the current file and the superseded file is no longer served (EARS-2).
- Every absolute date/time is stored as **one canonical instant** and rendered in `Europe/Moscow` labeled МСК, on the admin surface and in every produced projection (EARS-10).
- Every 007 command and admin read requires the `platform_admin` role; a `doctor_guest` or public caller is **never** admitted to author, transition, or read admin data (EARS-8).
- Each lifecycle transition appends **exactly one** terminal `audit_ledger` row (EARS-4, EARS-5, EARS-6; ADR-0003 §6).
- `ended → archived` is a **manual** operator action — no scheduler fires it in wave 1 (EARS-6, LD-2).

## Dependencies & sequencing

- **007 is the producer that closes the wave-1 seams.** 004 (event page/listing), 005 (registration), and 006 (room) were built + E2E-driven on **seeded events + seeded stream config + a seeded roster** because 007 had not landed; 007 lands the **real authored aggregate + lifecycle + stream config**. The "done against the real dependency" criterion for those seams is _"the 004/005/006 slices run against events authored, transitioned, and stream-configured through the 007 admin app, not seeds"_ — carried on the 004↔007 / 005↔007 / 006↔007 blocking links (`open-ears-issues` step 4). 007 is the producing side of the exact seams 005/006 named from the consuming side.
- **MVP-critical deadline — 2026-07-17.** The minimal path create → publish → configure stream → open → close must be completable by one operator/director for the live webinar, without developer intervention (PRD acceptance). 007's happy-path E2E is that arc in the admin app.
- **Auth foundation (feature 003) — shipped.** The admin principal is a `platform_admin` session from the shipped 003/IdP layer; 007 adds no auth primitive (EARS-8). The Refine admin app is 2FA-gated (ADR-0004 §3).
- **Object storage (Timeweb Object Storage / MinIO dev stand).** The program PDF is uploaded to object storage; the e2e upload test runs against the dev-stand MinIO (`S3_ENDPOINT`), `skipIf` absent — endpoints/bucket read from `.env.local`, never hardcoded (`.claude/rules/dev-stand.md`; ADR-0003).
- **Public event page + listing (feature 004).** 004 renders the produced `PublicEventPage`/`UpcomingBroadcastCard`; 007 authors the data behind them and owns the lifecycle transitions 004 reads (`EventPublished`/`WentLive`/`Ended`/`Archived`). Publishing/archiving through 007 is what flips 004's public reachability off seeds onto the real aggregate.
- **Registration (feature 005).** 005 gates the register affordance on the `EventLifecycleState` 007 writes (published/live offered; ended/archived withheld). 007's publish opens registration; 007 owns no registration record.
- **Webinar room (feature 006).** 006 reads the 007 stream config (provider enum + embed ref) to instantiate the player and gates admission on `live`; **the 006 open/close it named as its 007 seam are the 007 `OpenRoom`/`CloseRoom` commands** (EARS-5). 006's "done against the real dependency" criterion — the room opens/closes via 007 director controls and instantiates from 007-authored config — is satisfied by this feature.
- **Admin framework (Refine, ADR-0004).** The admin surface is stock Refine + custom data/auth/access providers over the NestJS API; no admin canvas exists (Stage-A gap) so no design-system canvas dependency applies. The `build-ui-from-design-system` adopt-before-bespoke gate runs for any bespoke element; ADR-0013 token discipline applies where custom styling appears (EARS-11).
- **Dev-stand Postgres + Zitadel + MinIO.** The admin e2e tests run against dev-stand Postgres (`DATABASE_URL`) + the live stand `idp` (`IDP_ISSUER`, for the `platform_admin` session) + MinIO (`S3_ENDPOINT`) per `.claude/rules/dev-stand.md`; endpoints/ports read from `.env.local`.
