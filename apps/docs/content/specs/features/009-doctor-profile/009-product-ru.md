## Статус: Суперседнута → #770

Read-only профиль врача, который скоупил этот PRD, **доставлен через [#770](https://github.com/doctor-school/ds-platform/issues/770)**, а не через EARS-триплет 009. `/account` теперь реальный продуктовый профиль — постерный хедер, секции Профиль / Безопасность / Сессия, inline-редактирование имени, email + бейдж verified, телефон («не указан», когда отсутствует) и выход — питаемый `GET /v1/me/profile` (фича 003 **EARS-27 / EARS-28**), заменивший сырой дамп session-claims. Вертикальный deliverable (реальный аутентифицированный read-эндпоинт своего профиля + продуктовая поверхность), которого требовал этот PRD, отгружен.

**Не пишите EARS-триплет из этого PRD.** Он сохранён только как запись решения, чтобы декомпозиция эпика 008/009 оставалась читаемой; живое поведение принадлежит #770 и фиче-003 EARS-27/28.

### Остаток на проверку (не новый scope)

- **MFA / состояние безопасности (бывший US-2).** Этот PRD просил профиль описывать состояние безопасности врача продуктовым языком. Отгруженная секция «Безопасность» на `/account` показывает «Сменить пароль»; показывается ли явный **индикатор MFA вкл/выкл**, не подтверждено против отгруженной страницы. Если продуктовый владелец хочет этот индикатор — это **небольшой follow-up на отгруженном профиле (поверхность #770)**, оформленный отдельной Issue, а не возрождение фичи 009.

_Суперседнута 2026-07-15 (решение владельца). Оболочка портала, которая ведёт на `/account`, — фича **008** (восстановлена вместе с этим указателем); профиль за ссылкой — #770._
