> **RU (это)** · **EN:** [`brief.md`](./brief.md)

> **Scope (решение владельца, брейншторм 2026-07-04).** Эпик покрывает **только вебинары — онлайн-эфиры**. Конгрессы (переданы внешней команде), офлайн-мероприятия и платные тарифы — **явно вне** эпика; каждое — будущий отдельный эпик. Цель: живой вебинар **2026-07-17** проходит на новой платформе end-to-end.

---

## Проблема

- Вебинарный бизнес Doctor.School работает на легаси-стеке Bubble — медленном, размещённом в США, троттлящемся из-за РКН, с 3 годами техдолга — это и есть драйвер пересборки (recon §7a).
- **Пайплайн присутствия отключён с 2026-04-29** — ключевой B2B-результат (минуты присутствия по каждому врачу для спонсора) для новых вебинаров сейчас вообще не производится (recon §6).
- Врачи регистрируются и затем **не могут найти мероприятие, на которое записались**; гонки логин-сессий дают жалобы «не смог подключиться к эфиру» в день эфира (recon §7c, §7e).
- Следующий спонсируемый живой вебинар назначен на **2026-07-17** и должен пройти на новой платформе.

## Jobs-to-be-done

- **Фарм-спонсор** (платит за всё): получить **верифицируемый охват таргетированной по специальности врачебной аудитории** — отчёт с контактами + фактическими минутами присутствия по каждому врачу.
- **Врач:** найти релевантный эфир, зарегистрироваться в пару кликов, смотреть вживую и общаться в чате; позже — получать баллы НМО автоматически.
- **Оператор / режиссёр:** завести мероприятие за ≥1 месяц; в день эфира вставить ссылку на трансляцию и открыть комнату; после — закрыть её и передать данные посещаемости.

## Сквозная информационная архитектура

- **Все зрительские поверхности живут в приложении `portal`** (личный кабинет). Страница мероприятия **публично читаема** — спонсоры распространяют прямые ссылки по своим каналам; так и работает discovery. CTA «Участвовать» идёт через auth (фича 003, уже отгружена).
- **Вебинарная комната закрыта на сервере:** только аутентифицированные + зарегистрированные — никакой мягкой UI-стены поверх плеера в легаси-стиле.
- **Админ-поверхности живут в приложении `admin`** (Refine, ADR-0004).
- **Жизненный цикл мероприятия — единая state machine** — `draft → published → live → ended → archived` — а не легаси-россыпь булевых флагов.
- **Вебинарную комнату строим сами как композицию**, где видео — **внешний стрим (Rutube или аналог), встроенный как сконфигурированный iframe/плеер** — без видеотранскодинга, без собственного медиасервера. Buy-вариант Mediator.cloud рассмотрен и **отклонён**.
- **Присутствие фиксируется с первого дня** через **server-authoritative heartbeat** (аутентифицированный POST каждые N секунд → append-таблица в нашем Postgres). Отчёт спонсору по первому вебинару — **ручная выгрузка** из этих данных; авто-НМО и авто-отчёт — волна 2.

## Декомпозиция фич

Утверждён вариант **A «тонкая вертикаль»** — четыре фичи волны 1, все `surface: user-facing`, у каждой ко-локализованный PRD:

| #   | Фича                                                                              | В одну строку                                                                                                                                                           |
| --- | --------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 004 | [`event-page-listing`](../../features/004-event-page-listing/004-product-ru.md)   | Публичная страница мероприятия (до эфира: hero, дата/время, спикеры, PDF программы, специальности, партнёры; CTA регистрации) + минимальный листинг ближайших эфиров.   |
| 005 | [`event-registration`](../../features/005-event-registration/005-product-ru.md)   | Регистрация поверх auth 003 (залогинен — one-tap; гость → auth-флоу → регистрация завершается), видимость «мои события», состояние регистрации на странице мероприятия. |
| 006 | [`webinar-room`](../../features/006-webinar-room/006-product-ru.md)               | Аутентифицированная комната: embed-плеер (явный enum провайдера), живой чат (Centrifugo), heartbeat-фиксация присутствия, питающая отчёт спонсору.                      |
| 007 | [`event-admin-minimal`](../../features/007-event-admin-minimal/007-product-ru.md) | Админка/Refine: создание/редактирование мероприятия, конфиг ссылки на стрим, открытие/закрытие живой комнаты, переходы состояний мероприятия.                           |

**Волна 2** (будущие фичи этого эпика — папок PRD пока нет):

- Календарь + фасеты по специальностям в листинге.
- Опросы + вопрос лектору в комнате.
- Автоначисление НМО (90 минут + 2 подтверждения присутствия).
- Автоматический отчёт спонсору «Отчёт партнёра V2».
- Уведомления (welcome / подтверждение / напоминания).
- Архив записей.

**Полностью вне эпика** (будущие отдельные эпики): конгрессы, офлайн-мероприятия, платные тарифы.

## Метрики успеха

- **Живой вебинар 2026-07-17 проходит end-to-end на новой платформе**: реальные регистрации врачей → живой просмотр → присутствие записано.
- **Спонсор первого вебинара получает отчёт с минутами присутствия** (ручная выгрузка допустима).
- **Ноль жалоб «не смог подключиться к эфиру» по причинам на стороне платформы** (легаси-класс гонок сессий).

## Prior art — система-источник

Легаси-система Doctor.School (Bubble + Directual + Supabase + бизнес-база знаний + живой сайт) отмайнена read-only 2026-07-02; полное функциональное свидетельство — доменная модель, функциональная карта по поверхностям, механика стрима/realtime, пайплайн присутствия, боли, которые надо превзойти, и реестр открытых вопросов — живёт в [`legacy-recon.md`](./legacy-recon.md) ([RU](./legacy-recon-ru.md)). В одном абзаце: вебинар — это **спонсируемый эфир для врачей, таргетированных по специальности**, чей B2B-результат — отчёт о посещаемости по каждому врачу; легаси-реализация доказывает каждую возможность, нужную этому эпику (регистрация, встроенный внешний плеер, живой чат, фиксация минут присутствия, начисление НМО), и одновременно доказывает ошибки, которые надо превзойти — 130-полевой агрегат события с лайфсайклом на россыпи булевых флагов, клиентские пинги присутствия с засвеченным сервисным ключом (отключены с 2026-04-29), выбор плеера по URL-sniffing и «realtime» без push. Разведка — референс «смотри-и-бери-домен», но никогда не целевая схема и не UI (ADR-0014 §3).

### Вердикты adopt-vs-build (разведка 2026-07-04)

| Компонент                                       | Вердикт                                | Обоснование + источник                                                                                                                                                           |
| ----------------------------------------------- | -------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Вебинарный фреймворк целиком                    | **BUILD** (как композиция)             | plugNmeet / BigBlueButton / La Suite Meet — WebRTC-стеки с собственными медиасерверами, что противоречит embed-рамке. (plugnmeet.org, github.com/suitenumerique/meet)            |
| Realtime-канал (чат, join/leave, позже виджеты) | **ADOPT** Centrifugo                   | Уже в нашем стеке; per-channel presence, presence_stats, join/leave, history. (centrifugal.dev/docs/server/presence)                                                             |
| UI чата                                         | **ADOPT** shadcn                       | Официальные чат-примитивы (ui.shadcn.com changelog 2026-06) или MIT shadcn-chat (shadcn-chat.vercel.app); финальный выбор на этапе delivery через `build-ui-from-design-system`. |
| Хранилище присутствия                           | **BUILD** (маленький)                  | Presence Centrifugo эфемерен; отчёту спонсору нужны долговечные минуты → 1 аутентифицированный эндпоинт + 1 append-таблица.                                                      |
| Embed-плеер                                     | **ADOPT** react-player / тонкий iframe | react-player для YouTube (github.com/cookpete/react-player); тонкий iframe для Rutube; **явный enum провайдера, никогда не URL-sniffing**.                                       |
| UI календаря / листинга                         | **ADOPT** shadcn event-calendar blocks | На этапе delivery (shadcn-event-calendar.vercel.app и т.п.); волне 1 нужен только минимальный листинг.                                                                           |
| Режиссёрский пульт / админка                    | **ADOPT** Refine                       | Уже стек админ-приложения (ADR-0004).                                                                                                                                            |
